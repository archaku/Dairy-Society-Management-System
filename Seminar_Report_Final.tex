\textbf{ABSTRACT}
\end{center}

\vspace{1cm}
\textcolor{red}{}\\
The Dairy Society Management System is a web-based platform designed to digitalize and streamline traditional dairy society operations. It aims to eliminate the inefficiencies of manual recordkeeping and bring greater transparency to essential processes such as milk collection, milk quality assessment, farmer payments, and product distribution. By centralizing all records in a digital environment, the system ensures faster data access, reduces documentation errors, and creates a reliable communication channel between farmers, customers, and society administrators. The overall workflow becomes more organized, accurate, and time-efficient, supporting better coordination across daily operations.

The application incorporates three major functional modules: Farmer, Society Administrator, and Customer. Through the Farmer Module, farmers can record their daily milk supply, track quality-based payment calculations, view transaction and settlement history, and receive automated stock notifications for cattle feed and supplements. Furthermore, farmers are empowered to list their daily stock surplus for direct customer pre-booking and automated long-term subscriptions. The Society Administrator Module offers tools to manage milk collection records, verify daily entries, generate farmer payouts, manage profiles, and monitor overall milk inventory using dynamic dashboards and analytics reports. These features significantly reduce the administrative burden, improve accuracy, and enable faster decision-making based on real-time data.

In addition, the system includes a Customer Module where users can purchase milk directly from the society or bypass traditional supply chains entirely through a direct farmer-to-customer sales pipeline. Through a simple online ordering interface, customers can view live local stock availability, place daily pre-booking orders, and establish recurring daily milk subscriptions seamlessly linked to secure payment gateways. By integrating modern automation and structured data management, the platform modernizes dairy workflows and strengthens the operational backbone of agricultural societies. It enhances transparency, minimizes manual errors, and promotes better financial management for farmers. Overall, the system creates a scalable and efficient digital ecosystem that benefits farmers, customers, and administrators alike, contributing to a more sustainable and reliable dairy supply chain.

\chapter{INTRODUCTION}
\section{Introduction to the Project}
\vspace{0.5cm}
The agricultural dairy sector is fundamentally transitioning from localized, paper-based manual ledgers to predictive, structurally robust distributed digital architectures. The Dairy Society Management System (DSMS) targets exactly this paradigm shift. By incorporating Node.js asynchronous handling and the React DOM, it allows cooperative societies to handle simultaneous operations—such as daily milk logging from hundreds of farmers, real-time quality metric evaluations, and continuous pre-booking requests from customers—without latency or data collisions.

Historically, traditional cooperative operations involved massive synchronous paperwork and disconnected spreadsheets, leading to unfair pricing, erroneous ledger balances, and severe payment delays. The core strategy underpinning DSMS is a structurally unyielding MongoDB non-relational database combined with robust Express.js API pipelines, ensuring highly robust throughput. From the farmer's perspective, this results in transparent, immediate insights into their daily milk yield, quality benchmarking (Fat, SNF, and pH metrics), and real-time financial balances.

Furthermore, integrating advanced structural features isn't simply a buzzword—it provides tangible operational benefits. Rather than relying on rigid supply chains, DSMS introduces a dedicated Direct Milk Sales pipeline. It allows customers to pre-book milk directly from local farmers, bypassing processing bottlenecks. When customers opt for long-term supply, secure financial connections via the Razorpay Payment Gateway seamlessly activate Subscription workflows. Localized inventory mechanisms are automated accurately.

Ultimately, DSMS guarantees that any modern device—be it a legacy smartphone utilized by a farmer in a rural network area or a cooperative administrator's desktop—receives completely optimized HTML payloads via the React DOM engine. This holistic approach proves that grassroots agricultural authorities can adopt nimble, responsive technology stacks mirroring top-tier commercial e-commerce platforms.
\vspace{0.5cm}

DSMS is an advanced web-based platform designed to revolutionize dairy cooperative operations. The system provides an end-to-end centralized architecture for tracking daily milk collection, verifying quality markers instantly, automating transparent payments via Razorpay, and connecting farmers directly to consumers for subscription-based purchasing. By focusing on modern responsive web development standards, it directly addresses limitations inherent in legacy agrarian interfaces.

\chapter{REQUIREMENTS AND ANALYSIS}
\section{Existing System / Problem Statement}
Current regional dairy cooperative systems often suffer from visually poor interfaces (or complete lack thereof), rigid monolithic cash-entry accounting, and massive delays in performance feedback for suppliers. Farmers find it difficult and extremely time-consuming to dispute quality metrics (SNF/Fat), while customers struggle to access fresh, locally sourced dairy without passing through multiple commercial processing layers heavily impacting freshness. In conventional setups, tracking individual milk contributions and executing fair payments remains a severe bottleneck for cooperative authorities.

\section{Proposed System / Solution Overview}
DSMS functions as a complete modernization alternative. It acts as a dynamic agricultural engine. It actively reads the live database to calculate strict ledger updates mathematically, immediately logging daily milk records virtually while assuming instantaneous payment cycle metrics. Additionally, advanced analytics and custom charting dynamically dictate the "Performance" arrays, evaluating farmer yields over customized timelines. Razorpay handles precise web payment verifications, while JWT constructs secure boundaries minimizing architectural fraud natively. It empowers Direct Selling, linking consumers directly to farmers.

        // Filter out any availabilities where the farmer no longer exists
        availabilities = availabilities.filter(avail => avail.farmer !== null);

        const adjustedAvailabilities = [];
        for (let avail of availabilities) {
            // Subtract Subscriptions
            const activeSubs = await MilkSubscription.find({
                farmer: avail.farmer._id,
                shift: avail.shift,
                status: 'active',
                startDate: { $lte: new Date(targetDate.getTime() + 86400000 - 1) },
                endDate: { $gte: targetDate }
            });
            
            const subscribedQty = activeSubs.reduce((sum, sub) => sum + sub.quantityPerDay, 0);

            // Subtract Direct Sales (Pending)
            // Use availability date start/end to avoid timezone issues
            const availStart = new Date(avail.date);
            availStart.setHours(0,0,0,0);
            const availEnd = new Date(availStart);
            availEnd.setHours(23,59,59,999);

            const pendingSales = await DirectMilkSale.find({
                farmer: avail.farmer._id,
                shift: avail.shift,
                status: 'pending',
                date: { $gte: availStart, $lte: availEnd }
            });
            const pendingQty = pendingSales.reduce((sum, sale) => sum + sale.quantity, 0);

            const remainingQty = avail.availableQuantity - subscribedQty - pendingQty;

            if (remainingQty > 0) {
                adjustedAvailabilities.push({
                    ...avail.toObject(),
                    availableQuantity: Math.max(0, remainingQty),
                    originalQuantity: avail.availableQuantity
                });
            }
        }

        res.json({ success: true, availabilities: adjustedAvailabilities });
    } catch (error) {
        res.status(500).json({ success: false, message: error.message });
    }

            date: { $gte: availStart, $lte: availEnd }
        });

        if (remainingQty < quantity) {

                message: `Insufficient milk available. Stock reserved for subscriptions and pending requests leaving only ${remainingQty.toFixed(1)}L open.`
            });

            await availability.save();

            shift,
            date: targetDate

const verifyUser = async (req, res, next) => {
    try {
        const token = req.headers.authorization?.split(' ')[1];
        if (!token) return res.status(401).json({ success: false, message: 'No token' });
        const decoded = jwt.verify(token, process.env.JWT_SECRET);
        if (decoded.role !== 'user') return res.status(403).json({ success: false, message: 'User access only' });
        req.userId = decoded.userId;
        next();
    } catch (err) { res.status(401).json({ success: false, message: 'Invalid token' }); }
};

// @desc    Create a new subscription (User)
router.post('/', verifyUser, async (req, res) => {
    try {
        const { 
            farmerId, 
            quantityPerDay, 
            startDate, 
            endDate, 
            shift, 
            distance, 
            deliveryLocation,
            razorpay_payment_id,
            razorpay_order_id,
            razorpay_signature
        } = req.body;

        // Verify Razorpay Payment Signature
        const body = razorpay_order_id + "|" + razorpay_payment_id;
        const expectedSignature = crypto
            .createHmac('sha256', process.env.RAZORPAY_KEY_SECRET)
            .update(body.toString())
            .digest('hex');

        if (expectedSignature !== razorpay_signature) {
            return res.status(400).json({ success: false, message: 'Invalid payment signature' });
        }

        const farmer = await Farmer.findById(farmerId);
        if (!farmer) return res.status(404).json({ success: false, message: 'Farmer not found' });

        if (!farmer.offersSubscription) {
            return res.status(400).json({ success: false, message: 'This farmer is not currently accepting new subscriptions.' });
        }

        if (distance > farmer.subscriptionDeliveryRange) {
            return res.status(400).json({ success: false, message: `Delivery distance (${distance} km) exceeds the farmer's maximum range of ${farmer.subscriptionDeliveryRange} km.` });
        }

        const start = new Date(startDate);
        const end = new Date(endDate);
        const diffTime = Math.abs(end - start);
        const days = Math.ceil(diffTime / (1000 * 60 * 60 * 24)) + 1; // inclusive

        if (days <= 0) return res.status(400).json({ success: false, message: 'Invalid date range' });

        const pricePerLiter = farmer.subscriptionMilkRate || 50;
        const milkTotal = pricePerLiter * quantityPerDay * days;
        const deliveryCharge = distance * 15 * days;
        const totalAmount = milkTotal + deliveryCharge;

        const subscription = new MilkSubscription({
            user: req.userId,
            farmer: farmerId,
            quantityPerDay,
            startDate: start,
            endDate: end,
            shift,
            pricePerLiter,
            deliveryCharge,
            totalAmount,
            distance,
            deliveryLocation,
            paymentStatus: 'Completed',
            paymentId: razorpay_payment_id,
            status: 'active'
        });

        await subscription.save();
        res.status(201).json({ success: true, subscription });
    } catch (error) {
        res.status(500).json({ success: false, message: error.message });
    }
});

router.get('/farmer/today', verifyFarmer, async (req, res) => {
    try {
        const today = new Date();

        // Find active subscriptions where today is between start and end date
        const activeSubs = await MilkSubscription.find({
            farmer: req.farmerId,
            status: 'active',
            startDate: { $lte: new Date(today.getTime() + 86400000 - 1) }, 
            endDate: { $gte: today }
        }).populate('user', 'firstName lastName phone address');

        // Check which ones have delivery logs for today
        const deliveriesToday = await SubscriptionDelivery.find({
            subscriptionId: { $in: activeSubs.map(s => s._id) },
            date: { $gte: today, $lt: new Date(today.getTime() + 86400000) }
        });

        const deliveryMap = {};
        deliveriesToday.forEach(d => {
            deliveryMap[d.subscriptionId.toString()] = d;
        });

        const schedule = activeSubs.map(sub => {
            const log = deliveryMap[sub._id.toString()];
            return {
                subscription: sub,
                deliveryStatus: log ? log.status : 'pending',
                deliveryLogId: log ? log._id : null
            };
        });

        res.json({ success: true, schedule });
    } catch (error) {
        res.status(500).json({ success: false, message: error.message });
    }
});
